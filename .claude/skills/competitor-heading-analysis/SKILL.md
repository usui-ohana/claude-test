# 競合見出し分析

検索上位の競合記事がどのような見出し構成（情報設計）になっているかを定量的に把握し、自分の記事の見出し構成を「競合の網羅性に負けない」かつ「競合が触れていない切り口を補える」ように設計するためのスキル。

## 全体の流れ

1. **競合記事のURLを特定する。**
   - ユーザーから具体的なURLを渡された場合はそれを使う。
   - URLが渡されていない場合は、対象キーワードでWebSearchを行い、上位に出てくる記事のURLを数件（目安3〜5件）ピックアップする。検索結果から明らかに比較対象として不適切なもの（広告、ECの商品ページ単体など）は除く。

2. **各記事のHTMLを取得する。**
   - WebFetchツールでページのHTMLまたは本文を取得する。WebFetchが要約・整形したテキストしか得られない場合は、見出しらしき行（短い行、太字、番号付きの大項目など）を手がかりに見出し構造を推定してよいが、可能な限り生のHTMLから抽出する方が正確。

3. **見出し構造を抽出する。**
   - 生HTMLが手に入った場合は、必ず `scripts/extract_headings.py` を使って機械的に抽出する（目視でのタグ拾い出しは見落としやすいため避ける）。

   ```bash
   python scripts/extract_headings.py path/to/page.html
   # またはHTML文字列を標準入力で渡す
   echo "$HTML" | python scripts/extract_headings.py
   ```

   - 出力はH1〜H6のレベルと見出しテキストの一覧（Markdown表）。

4. **複数記事の見出しを横並びで分析する。**
   - 各記事の見出し一覧を並べ、次の観点で整理する。
     - **共通して扱われているトピック**: 複数の競合が触れている＝検索ユーザーが当然期待する情報。自分の記事でも外せない。
     - **一部の記事だけが扱っているトピック**: 差別化や網羅性強化のチャンスになりうる。
     - **どの競合にも無いが、検索意図から考えて欲しいはずの情報**: ここを補えると独自性が出る。
     - **見出しの順序・粒度**: 結論を先に出しているか、初心者向けの説明から入っているかなど、構成のパターンを把握する。

5. **自分の記事の見出し構成案を作る。**
   - 上記の分析結果をもとに、H2/H3の見出し構成案を提示する。
   - 競合の見出しをそのまま流用・コピーしない。トピックの「網羅すべき範囲」は参考にするが、見出しの文言・切り口は自分の記事の文脈に合わせて作り直す。
   - 各見出しについて、なぜそれを入れるのか（競合で共通して扱われている／差別化要素として有効／検索意図上欲しいはず、など）を簡潔に根拠付けできるようにしておく。

## 出力形式

- 分析した競合記事ごとの見出し一覧（簡潔でよい）
- 提案する見出し構成案（H2/H3の階層を明示したアウトライン）
- 各見出しを採用した理由の簡単な補足

## 注意点

- このスキルは見出しの「骨格」を作るための分析であり、本文の文章自体は生成しない。本文執筆は別のタスク（記事執筆フロー）に委ねる。
- 競合のタイトルや見出しの言い回しをそのまま転用すると独自性が薄れるだけでなく、表現の盗用にも見えうるため、必ず自分の言葉に言い換えること。
