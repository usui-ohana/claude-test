# 文字数カウンター

テキストの文字数を機械的に正確に数えるためのスキル。手計算や目視でのカウントは誤りやすいため、必ず付属スクリプトを使うこと。

## カウント対象の定義

- **スペース**: 半角スペース（` `）、全角スペース（`　`）、タブ（`\t`）の3種類のみを対象とする。
- **改行**: `\n` および `\r` を対象とする。
- 判定の詳細（コード値、CRLFの扱い、用途別の使い分け）は [references/counting_rules.md](references/counting_rules.md) を参照。閾値の意味で迷ったらこのファイルを読むこと。

## 出力する4パターン

| パターン | 内容 |
|---|---|
| 改行含む・スペース含む | 何も除去せずそのままの文字数 |
| 改行含む・スペース除く | 改行は残し、スペース類のみ除去 |
| 改行除く・スペース含む | 改行のみ除去し、スペース類は残す |
| 改行除く・スペース除く | 改行・スペース類の両方を除去 |

## 実行手順

1. **対象テキストを特定する。**
   - ユーザーがチャットに直接貼った文章 → そのままヒアドキュメントで渡す。
   - ファイルパスが示された／添付された場合 → そのファイルパスをスクリプトの引数に渡す（内容を自分で読み込んでコピー&ペーストしない。文字コードや改行コードが変化するリスクを避けるため、必ずファイルパスごと渡す）。
   - 対象が複数（例: 複数ファイル、または文章中の特定の段落だけ）の場合は、対象ごとに1回ずつスクリプトを実行し、結果を分けて提示する。
2. **必ず `scripts/count_chars.py` を実行してカウントする。** 文字数を目視・暗算・推測で答えることは禁止。スクリプトの実行結果のみを根拠とする。

**テキストを直接渡す場合:**
```bash
python scripts/count_chars.py <<'EOF'
ここに対象のテキストを貼る（複数行可、末尾の改行の有無もそのまま反映される）
EOF
```

**ファイルを渡す場合:**
```bash
python scripts/count_chars.py path/to/file.txt
```

3. **結果を提示する。** 基本はスクリプトが出力するMarkdown表をそのまま回答に含める（改変・四捨五入・要約はしない）。
   - 対象がファイルの場合、またはユーザーがレポート的な見せ方を求めている場合は、[assets/report_template.md](assets/report_template.md) のテンプレートに沿って整形する（`{{source}}` に対象名、`{{count_*}}` に各パターンの数値を埋める）。
4. **ユーザーが「○文字以内か」のような閾値を聞いている場合**は、4パターンのうちどの数え方が適切か（例: SNS投稿の文字数制限なら通常「改行含む・スペース含む」が実態に近い）を一言添えて、閾値との比較結果も明示する。
5. 入力が空文字や空行のみの場合でもスクリプトはエラーにならず0件としてカウントされる。スキップせず通常通り実行する。
